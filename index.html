What I hope to Get out of Web Foundations

I found the PCS JavaScript evening programs and quickly realized I didn't quite have the foundation to start off with one of those, and honestly I was a little leery of diving right in at that price tag, so the foundations series seemed like a good way to dip my toe in the water. What I would really like to get is this:

  1. An idea of whether my instinct about development as a career is right, and whether I am cut out for it.
  2. An idea of what I might expect out of either the Full Stack or Ruby On Rails evening courses.
  3. An idea of which course to take to be able to get an entry level development position.
  4. The skills to not be totally lost in either of those courses.
  5. Some friendly coding buddies would be a nice plus.

Concerns

  • I don't know any JavaScrip.
  • I know very little CSS.
  • I don't know how to make my command line work right.
  • I don't know how to use Git yet!
  • I only have windows PCs.

HTML

Glossary

This card collects terms and their definitions into a Description List Element (dl) for later reference.

tag name
The word inside an HTML tag that defines the element
tag
A tag name and opening and closing delimeters. An closing tag puts a forward slash in front of the tag name.
element
An opening tag, some content, and a closing tag. Some elements don't have closing tags and some elements are self closing.

CSS

CSS Positions

This article is about CSS positioning, which is finicky and tricky.

The position property can be used to determine what portion of the screen an element is tethered to:

position:static
This is the default positioning. An item will behave as you would normally expect, scrolling off the screen, etc.
position:relative
This allows an element to be positioned relative to its normal position using the top, bottom, left, and right properties.
position:fixed
Fixed position elements will be tethered to a spot on the screen even if the screen is scrolled.
position:absolute

Git Glossary

These terms relate to git and github

working directory
the directory you are in (usually that you are about to add to your repository)
staging area
An area where git stores a list of files before making a commit
repo (repository)
A colletion of all your commits
Remote
A cloud stored version of your repository, usually on a service like Github. You push to and pull from your remote.
Branch
A branch is a thread of development.

Example: gh-pages on github is the thread that is live on the web server.
